智能合约

先来看智能合约部分。

拍脑袋自己想的足彩智能合约需要的功能：

  1. 庄家发起游戏，需要保证金
  2. 玩家下注参与游戏
  3. 在比赛结束后输入结果
  4. 玩家 输掉 提现
  5. 庄家回首本金和收益

而一场足球比赛进行菠菜需要：

  1. 主队和客队
  2. 开赛时间，结束时间
  3. 赛果 主胜 / 平局 / 客胜
  4. 赛果对应利率

确定好之后打开你的编辑器，开始编写智能合约吧！

pragma solidity ^0.4.19;

contract BetOnEther {

    // 不要吐槽我的命名
    
    address public host;      // 庄家账户。address类型是账号的地址。
    uint public earnestMoney; // 庄家保证金，限制游戏最大金额总量
    uint[3] public oddss;     // [win draw lose] 对应赔率
    bool public ended;        // 游戏是否已结束

    // 比赛结果 枚举
    enum Result { Win, Draw, Lose }

    // 比赛信息
    struct Game {
        bytes32 home;      // 主队名
        bytes32 visiting;  // 客队名
        bytes32 remarks;   // 备注 比如开赛时间，赛事系列
        uint result;       // 比赛结果
    }

    // 公开game信息
    Game public game;
    
    /// 创建
    constructor () public {}
    
    /// 下注
    function bet () public {}
    
    /// 输入比赛结果
    function confirm () public {}
    
    /// 提款
    function withdraw () public {}
    
    /// 庄家收钱
    function clear () public {}
}

public 属性表示可以被随意访问。在合约部署之后也可以很通过类似contract.host直接获取到保存的host信息。

当然以上的代码肯定是不能跑的，而且还缺少一些必要的东西：

  • 一个“奖金池”的概念，方便后面提现；
  • 记录用户下注的信息，并且用一个mapping将用户的账号和下注信息对应，也要支持一个用户多次下注。
  • 时间信息，比如下注截止时间，比赛结束时间等。

添加后合约如下：

pragma solidity ^0.4.19;

contract BetOnEther {

    // 不要吐槽我的命名
    
    address public host;      // 庄家账户
    uint public earnestMoney; // 庄家保证金，限制游戏最大金额总量
    uint[3] public oddss;     // [win draw lose] 对应赔率
    bool public ended;        // 游戏是否已结束

    // 比赛结果
    enum Result { Win, Draw, Lose }

    // 比赛信息
    struct Game {
        bytes32 home;      // 主队名
        bytes32 visiting;  // 客队名
        bytes32 remarks;   // 备注 比如开赛时间，赛事系列
        uint result;       // 比赛结果
    }

    // 公开game信息
    Game public game;
    
    
    // WHAT'S NEW
    
    bytes32 outerId;          // 用来对应我在链外database保存的信息
    uint public pool;         // 总奖金池
    uint public balance;      // 庄家收益
    uint[3] public bonuss;    // [w d l] 对应总奖金，任何一项超出保证金，则不能继续下注。
    
        // 下注
    struct Bet {
        uint result;     // 比赛结果
        uint stake;      // 金额
        uint odds;       // 赔率
        uint profit;     // 预计获利
        bool withdrawed; // 是否已经提款
    }

    // 所有玩家和下注的mapping
    mapping (address => Bet[]) public bets;
    
    uint public betEnd; // 下注结束时间
    uint public gameEnd; // 比赛结束时间
    
    function ...
}

创建合约

首先要明确，智能合约是建立在以太坊交易上的，每一次与智能合约的交互都是一笔带data的交易。

创建合约需要较多的参数，以及必须输入一定数量的以太币，作为庄家的保证金。solidity提供了payable熟悉，用来确保某次函数调用（一笔交易）必须有转账金额。

    constructor (bytes32 _home, bytes32 _visiting,
                 bytes32 _remarks, uint[3] _oddss,
                 uint _betTime, uint _gameTime,
                 bytes32 _outerId) public payable {
        host = msg.sender;
        oddss = _oddss;
        outerId = _outerId;

        betEnd = now + _betTime;
        gameEnd = betEnd + _gameTime;

        // 利润
        earnestMoney = msg.value;
        balance = earnestMoney;

        // 比赛信息
        game.home = _home;
        game.visiting = _visiting;
        game.remarks = _remarks;
    }

这是合约的构造函数，一些地方会用跟合约同名的函数来表示，也是可以的，但是remix会打出warning提示使用constructor。

msg.sender是交易的发起人，如同前面说的，每次函数调用都是一笔交易，msg就是保存了这笔交易信息的对象。常用的还有msg.value，表示交易金额。

这里的balance并不是一个很准确的信息，在合约执行中并没有太大的参考意义，我只在后面庄家提现的时候会用到它。

下注、提现、庄家收款

前面说过了payable，这里首先来看modifier，modifier简单来讲就是允许自己定义类似的属性，很像装饰器的概念。

    // 只允许庄家执行
    modifier onlyHost() {
        require(msg.sender == host);
        _;
    }

    // 只有在某个时间点后
    modifier onlyBefore(uint _time) {
        require(now < _time);
        _;
    }

    // 只有在某个时间点前
    modifier onlyAfter(uint _time) {
        require(now > _time);
        _;
    }

    // 游戏未结束
    modifier notEnd() { 
        require (!ended); 
        _; 
    }

require将会判断是否为真，否则直接跳出不会执行后面的代码。
now也是一个内建函数，将会得到当前的timestamp。

接下来就可以来定义三个接口，代码实现都是比较简单的。

    /// 下注
    function bet (uint _result) public payable onlyBefore(betEnd) {
        uint _profit = (msg.value * oddss[_result]) / 1000;

        // 对应总奖金不能超过保证金
        require ((bonuss[_result] + _profit) < earnestMoney);

        bets[msg.sender].push(Bet({
            result: _result,
            stake: msg.value,
            odds: oddss[_result],
            profit: _profit,
            withdrawed: false
        }));

        // 总奖池 + value
        pool += msg.value;

        // 对应总奖金 + _profit
        bonuss[_result] += _profit;
    }

    /// 输入比赛结果
    // function confirm (uint _result) public onlyHost onlyAfter(gameEnd) {
    function confirm (uint _result) public onlyHost notEnd {        
        game.result = _result;
        ended = true;

        // 利润 + 总奖池
        balance += pool;

        // 利润 - 开奖对应奖金
        balance -= bonuss[_result];
    }
    
    /// 提款
    // function withdraw () public onlyAfter(gameEnd) {
    function withdraw () public {
        require (ended);

        uint length = bets[msg.sender].length;
        uint amount = 0;
        
        for (uint i = 0; i < length; i ++) {
            var _bet = bets[msg.sender][i];
            if ((_bet.result == game.result) && !_bet.withdrawed) {
                amount += _bet.profit;
                _bet.withdrawed = true;
            }
        }

        if (amount > 0) {
            uint _amount = amount;
            amount = 0;
            msg.sender.transfer(_amount);
        }
    }

    /// 庄家收款
    // function clear () public onlyHost onlyAfter(gameEnd) {
    function clear () public onlyHost {
        require (ended);
        
        // 提取剩余利润
        uint _balance = balance;
        if (_balance > 0) {
            balance = 0;
            msg.sender.transfer(_balance);
        }
    }

运行一个私有链

我们需要用geth来运行一个私有链来做第一步测试。当然用truffle也是很不错的方式，本文代码最终开源到github上也是用truffle的结构的。这里不对truffle做详细介绍，只说明基本的创建私有链的方式，也是方便理解。

新建genesis.json，拷贝下列内容

// 创世块定义
{
    "config": {
        "chainId": 12345,
        "homesteadBlock": 0,
        "eip155Block": 0,
        "eip158Block": 0
    },
    "difficulty": "260",
    "gasLimit": "4700000",
}

找一个合适的文件夹，运行geth --datadir private init genesis.json
看到正确打印即是私有链创建好了。

运行geth --datadir private --networkid 12345 console 2>anywhereyoulike来看看这个区块链吧。

  • networkid是上面创世块中定义的
  • console表示打开一个用web3.jS和节点交互的console
  • 2>表示输出到别的地方，防止它影响我们的console。

进去之后执行eth.accounts，可以看到现在没有用户。先用personal.newAccount()创建一个用户。

用miner.start()开启挖矿。用eth.getBalance可以看到我们账户的金额正以惊人的速度增长（因为创世块的difficulty设置很低），这将作为我们之后的测试币。

ropsten

ropsten是以太坊的测试链，可以免费获取测试币，用来测试自己的智能合约。

连接到ropsten：
geth --datadir anywhereyoulike --testnet --syncmode light

  • --testnet 使用测试链，也可以用networkid 3，一样的效果，ropsten的id就是3，main是1。2是以前的测试链，现在不用了。
  • --datadir 可以不设置，mac默认目录是/Users//Library/Ethereum/testnet/。我习惯性会设置，需要注意的是，不要误删文件夹。当然也可以作为一了百了的手段。
  • --syncmode light 开启light同步模式。这种模式下，创世块直到最新区块之间的区块都会只同步一个head，可以用getBlock看看。大概几分钟就可以同步完，前提是有足够多的节点。ropsten的light模式找到节点的时间可能 一天都找不到 很久。

如果不用light模式，区块同步需要几个小时以及几十G的数据。
所以忍着吧，毕竟测试链不要钱。我的做法是在告诉用户当前节点是否可用。

web3.js

这是我刚接触智能合约时的部署方式，相信大部分人也一样。
这里介绍的是geth console（也就是web3.js交互方式）的方式部署，较多手动操作，看看就好，主要是熟悉一下remix的用法。

打开remix，首次打开会比较慢。

  1. 将刚才我们编写的代码全部拷贝进去。github上完整的代码。

  2. 看到右上角有一排导航栏，点击compile栏下的Start to compile。

  3. 不出意外remix将会打印出我们的错误并给出一些warning，都有很好的修改指引，有的话将其一一修改。

  4. 没有错误的话，点击编译按钮下面一行的Details，将会出现一个很长的弹出页，这有就是我们需要的部署合约的abi和bytecode。

  5. 找到WEB3DEPLOY，点击拷贝按钮，然后打开文本编辑器，黏贴。

  6. 因为我们合约的特殊性，构建函数是payable的，因此这种手动的方式，我们需要在构建时加点上交易金额的信息。var betonether = betonetherContract.new(...)就是合约的创建函数，做如下修改：

var betonether = betonetherContract.new(
   _home,
   _visiting,
   _remarks,
   _oddss,
   _betTime,
   _gameTime,
   _outerId,
   {
     from: web3.eth.accounts[0], 
     // 就是这里，填写要输入的保证金
     value: web3.toWei(10, 'ether'),
     ...
    }, function () {
     ...
    }
}）

当然像_home，_visiting这些变量，需要你在geth console中自己设定，可以从某些菠菜网站获取赛事信息和利率。

var betonetherContract = web3.eth.contract(...)
var betonether = betonetherContract.new(修改后的)

首先将你的账户解锁，然后将上面两行拷贝到geth console中。如果你正在挖矿，很快这个合约就将被写入区块中。

你可以创建第二个，第三个用户来作为玩家，调用betonether.bet()等函数来玩这个足彩游戏。
